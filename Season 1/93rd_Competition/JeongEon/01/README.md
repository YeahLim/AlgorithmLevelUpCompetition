# [백준 - 실버 1] 1309. 동물원

## ⏰**time**
10분

## :pushpin: **Algorithm**
DP (다이나믹 프로그래밍)

## ⏲️**Time Complexity**
$O(N)$

## :round_pushpin: **Logic**
1. 문제는 **2×N** 크기의 우리에 사자를 배치하는 경우의 수를 구하는 것이다.
   단, **세로 방향(같은 열)에 두 마리의 사자를 둘 수 없다**는 제약이 있다.
2. 각 열(칸)에 가능한 상태는 세 가지:
   * 사자 없음
   * 위에 사자 1마리
   * 아래에 사자 1마리
3. 따라서 점화식은 다음과 같이 유도된다:

```
D[0] = 1  (0열은 1가지 경우 - 아무것도 없음)
D[1] = 3  (사자 없음 / 위에 있음 / 아래에 있음)

D[i] = D[i-1] * 2 + D[i-2]
```

* D[i-1] * 2: 이전 칸에서 사자를 배치하지 않은 경우 또는 한 마리만 있는 경우 → 이번 칸에 사자 한 마리 추가 가능

* D[i-2]: i-1 칸에 사자를 배치하지 않은 경우 → i칸에 위/아래에 둘 수 있음

* **모든 연산은 `% 9901`**로 나머지 연산 수행

```
D[0] = 1
D[1] = 3
for i in 2 to N:
    D[i] = (D[i-1] * 2 + D[i-2]) % 9901
```

## :black_nib: **Review**
* 사자 배치 상태를 조합적으로 나누어 점화식을 세우는 게 핵심이었다.
* 경우를 나누고 전 칸의 상태를 기반으로 다음 칸을 세는 구조는 전형적인 DP 유형.
* 처음에는 3차원 배열로 상태를 관리하려 했으나, 단순한 수식으로도 해결 가능해서 놀랐음.
