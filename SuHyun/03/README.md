## :round_pushpin: **Logic**

1. 문제풀이

하스스톤 모독과 똑같은 메커니즘이다.

최대한 1,2,3,4,5 .. 와 같이 1의 차이가 나는 숫자열을 만들면 되는 문제이다.

이때 작은 숫자를 작게, 큰숫자를 크게 만드는 것이 가장 효율적이므로 오름차순으로 정렬하여 판단한다.
